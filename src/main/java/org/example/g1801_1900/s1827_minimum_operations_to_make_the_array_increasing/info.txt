Решение - жадный алгоритм

Подход: Это классическая задача, которая решается
жадным алгоритмом. Мы проходим по массиву, начиная со второго
элемента (индекс 1). Для каждого элемента мы проверяем, является
ли он строго больше предыдущего. Если нет, мы выполняем минимально
необходимое количество операций, чтобы это исправить.

Алгоритм:
Инициализируем счетчик операций operationsCount = 0.
Если длина массива меньше или равна 1, возвращаем 0.
Начинаем цикл с индекса i = 1 до конца массива.
На каждой итерации сравниваем текущий элемент nums[i] с предыдущим nums[i-1].
Если nums[i] > nums[i-1], переходим к следующему элементу.
Если nums[i] <= nums[i-1], это нарушает условие строгого возрастания.
Вычисляем, насколько нужно увеличить текущий элемент: requiredValue = nums[i-1] + 1.
Вычисляем, сколько операций для этого требуется: operationsNeeded = requiredValue - nums[i].
Увеличиваем счетчик операций: operationsCount += operationsNeeded.
Важный шаг: Немедленно изменяем значение текущего элемента в
нашем "мысленном" массиве на requiredValue. Это необходимо
для корректной проверки следующего элемента. Мы не меняем
исходный массив, но используем это новое значение для последующих сравнений.
