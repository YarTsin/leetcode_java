Объяснение подходов
Однопроходное решение (самое эффективное)
Принцип работы:
Инициализация: Начинаем с первого элемента как start и end
Итерация: Проходим по массиву, проверяя последовательность
Проверка последовательности: Если текущее число = end + 1, расширяем диапазон
Завершение диапазона: При разрыве последовательности, добавляем текущий диапазон
Форматирование: Диапазон форматируется в зависимости от длины

Ключевые моменты:
Используем переменные start и end для отслеживания текущего диапазона
Вспомогательный метод addRange для чистоты кода
Обработка последнего диапазона после цикла


Решение с двумя указателями
Принцип работы:
Внешний цикл: Проходим по началам диапазонов
Внутренний цикл: Находим конец последовательных чисел
Добавление диапазона: Форматируем и добавляем найденный диапазон
Переход: Перескакиваем к следующему непокрытому элементу


Преимущества однопроходного решения:
Более простая логика
Меньше вложенных циклов
Легче для понимания и отладки
Та же временная сложность

Особенности обработки:
Важно правильно обрабатывать пустой массив
Необходимо аккуратно работать с граничными значениями (MIN_VALUE, MAX_VALUE)
Форматирование строк должно быть корректным для отрицательных чисел

Сравнение подходов:
Однопроходный: Более линейная и простая логика
Два указателя: Более явное разделение на диапазоны
Эффективность: Оба подхода O(n) по времени и O(1) по дополнительной памяти
