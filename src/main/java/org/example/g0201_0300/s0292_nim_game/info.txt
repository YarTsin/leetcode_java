Объяснение:
Базовый случай: если камней 1-3, вы берете все и побеждаете
Если камней 4: любой ваш ход оставляет оппоненту выигрышную позицию

Общий паттерн: если n кратно 4, вы в проигрышной позиции,
иначе можете свести игру к кратной 4 для оппонента

========

Объяснение DP подхода:
Создаем массив dp, где dp[i] показывает, можно ли выиграть при i камнях
Базовые случаи: 1-3 камня - всегда true
Для каждого i проверяем, есть ли ход, после которого оппонент окажется в проигрышной позиции

========

Сравнение подходов
Математический подход:
Время: O(1)
Память: O(1)
Оптимальное решение

DP подход:
Время: O(n)
Память: O(n)
Полезен для понимания логики, но непрактичен для больших n
