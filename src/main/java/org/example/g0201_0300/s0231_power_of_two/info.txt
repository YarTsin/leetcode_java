Подробное объяснение
Проверка крайних случаев:

Все числа ≤ 0 автоматически не являются степенями двойки.

Битовый трюк:
Все степени двойки в двоичном представлении имеют ровно одну единицу:
1: 0001
2: 0010
4: 0100
8: 1000

Если вычесть 1 из степени двойки, все биты после единицы становятся 1:
4 (0100) - 1 = 3 (0011)

Побитовое И между числом и (числом - 1) даст 0 только для степеней двойки:
4 & 3 = 0100 & 0011 = 0000

==========

Преимущества этого решения
Эффективность по времени: O(1) - выполняются только несколько операций.
Эффективность по памяти: O(1) - не использует дополнительной памяти.
Простота: очень лаконичное и элегантное решение.
