Объяснение подходов
Решение с XOR (самое эффективное)
Принцип работы:
Операция XOR (исключающее ИЛИ) обладает следующими свойствами:
a ^ a = 0 (любое число, XOR с самим собой, дает 0)
a ^ 0 = a (любое число, XOR с 0, дает само число)

Операция коммутативна и ассоциативна: a ^ b ^ a = (a ^ a) ^ b = 0 ^ b = b

Как работает:
Все парные элементы при XOR дадут 0
Единственный элемент, не имеющий пары, останется после всех операций


Решение с HashSet
Использует дополнительную память для отслеживания уникальных элементов
Не удовлетворяет требованию константной памяти


Решение с сортировкой
Нарушает требование линейного времени из-за сортировки
Может быть полезно, если нет ограничений на время выполнения

Преимущества XOR подхода:
Абсолютно константная память (только одна переменная)
Линейное время выполнения
Простая и элегантная реализация
Не требует дополнительных структур данных


Важные особенности:
Решение работает только при условии, что все элементы, кроме одного, встречаются ровно дважды
Если бы элементы могли встречаться больше двух раз, это решение не подошло бы
Решение полагается на математические свойства операции XOR
Решение с XOR является оптимальным и полностью удовлетворяет всем требованиям задачи.
