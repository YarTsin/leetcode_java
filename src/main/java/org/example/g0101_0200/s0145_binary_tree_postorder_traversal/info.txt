Оптимальное решение (Итеративное с использованием двух стеков)
Объяснение подхода:
Используем два стека для имитации рекурсивного обхода
Первый стек обрабатывает узлы, второй хранит их в обратном порядке
Временная сложность: O(n) - каждый узел посещается один раз
Пространственная сложность: O(n) - для хранения двух стеков

Сравнение решений
Метод	Время	Память	Сложность реализации
Два стека	O(n)	O(n)	Средняя
Рекурсивный	O(n)	O(h)	Простая
Один стек	O(n)	O(h)	Сложная
Рекомендация: Для production-кода лучше использовать
итеративное решение с двумя стеками как наиболее читаемое
и эффективное. Рекурсивное решение проще, но может привести
к переполнению стека для больших деревьев.
