Что такое Preorder Traversal?
Preorder (прямой обход) - это способ обхода бинарного дерева, при котором:
Сначала посещаем корень
Затем посещаем все узлы левого поддерева
Затем посещаем все узлы правого поддерева

==========

Объяснение Morris Traversal
Morris Traversal - это алгоритм обхода бинарного дерева, который позволяет
обойти дерево in-order, pre-order или post-order без использования
стека или рекурсии, с постоянной (O(1)) дополнительной памятью.

Основная идея
Алгоритм временно модифицирует дерево, создавая временные связи
от правого потомка самого правого узла в левом поддереве обратно
к текущему узлу. Это позволяет "возвращаться" назад без использования стека.

Преимущества Morris Traversal
Память O(1) - не использует стек или рекурсию
Не разрушает дерево - временные связи удаляются
Эффективно - каждый узел посещается 2-3 раза

Недостатки
Сложность понимания - алгоритм нетривиален
Модификация дерева - временные изменения могут
быть проблематичны в многопоточных средах
Не читаемость - код сложнее для понимания и поддержки

Когда использовать?
Когда критически важна экономия памяти
В embedded systems с ограниченной памятью
На собеседованиях чтобы показать глубокое понимание алгоритмов

Для обычных задач
Рекурсивный или итеративный подходы предпочтительнее, так как:
Проще для понимания
Не модифицируют дерево
Достаточно эффективны для большинства практических задач

Morris Traversal - это скорее академический интерес,
демонстрирующий что можно достичь с помощью clever алгоритмов.
