Решение In-order обход с отслеживанием предыдущего значения

Основная идея:
In-order обход: Для BST in-order обход возвращает значения
узлов в отсортированном порядке.
Отслеживание предыдущего значения: Во время обхода сохраняем
значение предыдущего посещенного узла.
Вычисление разницы: Для каждого текущего узла вычисляем разницу
с предыдущим значением и обновляем минимум.

Почему это эффективно:
Сложность по времени: O(n) - мы посещаем каждый узел ровно один раз.
Сложность по памяти: O(h) - где h высота дерева (для
рекурсивного стека вызовов). В худшем случае (вырожденное дерево)
h = n, в среднем случае h = log(n).
