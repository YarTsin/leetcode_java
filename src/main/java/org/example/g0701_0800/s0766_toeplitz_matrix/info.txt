Последовательное сравнение элементов

Основная идея:
Для того чтобы матрица была матрицей Тёплица,
каждый элемент (кроме элементов последней строки и последнего
столбца) должен быть равен элементу, находящемуся от него
по диагонали вниз-вправо (matrix[i][j] == matrix[i+1][j+1]).

Таким образом, мы можем просто пройтись по всем элементам матрицы,
для которых существуют элементы [i+1][j+1]
(т.е., для всех i от 0 до m-2 и для всех j от 0 до n-2),
и проверить это условие. Если хотя бы одна пара не совпадает,
возвращаем false.

Почему это эффективно:
Сложность по времени: O(m * n). Мы посещаем каждый элемент
матрицы ровно один раз. Это оптимально, так как для ответа
нам нужно проверить каждый элемент как минимум один раз.

Сложность по памяти: O(1). Мы не используем дополнительную память,
пропорциональную размеру входа. Используются только несколько
переменных для итераций.
