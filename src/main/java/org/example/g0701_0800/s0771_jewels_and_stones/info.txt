Решение - Использование HashSet для драгоценностей

Основная идея:
Множество драгоценностей: Преобразовать строку jewels в HashSet
символов. Это позволит проверять, является ли камень драгоценностью,
за время O(1).
Подсчет камней: Пройти по каждому символу в строке stones
и проверить, содержится ли он в множестве jewels. Если да, увеличить счетчик.

Почему это эффективно:
Сложность по времени: O(n + m), где n - длина строки jewels,
m - длина строки stones.
O(n) на создание HashSet из jewels.
O(m) на проход по всем камням и проверку каждого в HashSet (O(1) на проверку).
Сложность по памяти: O(n) (для хранения множества драгоценностей).
В худшем случае, если все символы уникальны, n ≤ 50.
