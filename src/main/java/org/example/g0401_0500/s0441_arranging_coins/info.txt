Оптимальное решение (Бинарный поиск)
Объяснение подхода:
Используем бинарный поиск для нахождения максимального k
Вычисляем общее количество монет для k рядов: k*(k+1)/2
Если монет хватает, пробуем увеличить k
Если не хватает, уменьшаем k
Временная сложность: O(log n)
Пространственная сложность: O(1)
