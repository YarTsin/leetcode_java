Подходы к решению
1. Два указателя (оптимальное решение)
Используем два указателя: один для итерации, другой для записи уникальных элементов

Сложность O(n), память O(1)

2. Использование Set (не подходит по условиям)
Создаем Set для хранения уникальных элементов

Не подходит, так как требует дополнительной памяти O(n)

===============

Пояснение кода
Проверка пустого массива:

Если массив пуст, сразу возвращаем 0

Инициализация указателей:
uniquePointer - указывает на последний записанный уникальный элемент (начинается с 0)
currentPointer - итератор по массиву (начинается с 1)

Основной цикл:
Сравниваем текущий элемент с последним записанным уникальным
Если элементы разные:
Увеличиваем uniquePointer
Записываем новый уникальный элемент в позицию uniquePointer

Возврат результата:
Количество уникальных элементов = uniquePointer + 1 (так как индексация с 0)
