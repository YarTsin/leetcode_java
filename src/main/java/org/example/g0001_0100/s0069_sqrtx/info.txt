Подходы к решению
1. Линейный поиск (простой, но неэффективный)
Перебираем числа от 0 до x, пока квадрат числа не превысит x. Сложность O(n).

2. Бинарный поиск (оптимальный)
Используем бинарный поиск для нахождения корня, что дает сложность O(log n).

===========

Преимущества этого решения
Эффективность по времени: O(log n) - гораздо быстрее линейного поиска для больших чисел.

Эффективность по памяти: O(1) - используем только несколько переменных.

Защита от переполнения: использование long для хранения квадратов.
